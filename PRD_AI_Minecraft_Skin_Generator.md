# 产品需求文档：AI Minecraft 皮肤设计与生成工具

## 1. 文档信息

| 项目 | 内容 |
| --- | --- |
| 产品名称 | MCAISD - Minecraft AI Skin Designer |
| 文档版本 | v1.0 |
| 文档日期 | 2026-06-18 |
| 产品形态 | Web 应用优先，后续可扩展为桌面端或插件 |
| 目标平台 | PC 浏览器，移动端适配浏览和轻编辑 |
| 核心目标 | 让用户通过文字、参考图和可视化编辑，快速生成可用于 Minecraft 的个性化皮肤 |

## 2. 背景与机会

Minecraft 玩家常需要个性化皮肤来表达角色身份、服务器阵营、直播形象、IP 人设或社群统一风格。传统皮肤制作门槛较高，用户需要理解像素布局、角色模型、图层、四肢贴图方向等细节；直接搜索下载皮肤又容易撞款，且难以体现个人设定。

AI 生成皮肤工具可以降低创作门槛：用户只需要输入角色描述，系统即可生成符合 Minecraft 像素风和皮肤规格的 PNG，并提供 3D 预览、局部编辑、风格调整和一键导出能力。

## 3. 产品目标

### 3.1 业务目标

- 提升 Minecraft 皮肤创作效率，让普通玩家在 1-3 分钟内完成一个可用皮肤。
- 形成“生成 - 编辑 - 预览 - 导出 - 分享”的闭环工具体验。
- 支持免费试用和高级功能付费，为后续订阅、点数包、素材市场打基础。

### 3.2 用户目标

- 不懂像素画也能生成高质量 Minecraft 皮肤。
- 能把抽象描述转化为明确的角色形象。
- 能在生成结果基础上快速修改发型、服装、颜色、表情和配饰。
- 能导出符合 Minecraft 皮肤规格的 PNG 文件。

### 3.3 成功指标

| 指标 | MVP 目标 |
| --- | --- |
| 首次生成完成率 | >= 80% |
| 从输入描述到首张皮肤生成时间 | <= 60 秒 |
| 用户导出率 | >= 35% |
| 单用户平均生成次数 | >= 3 次 |
| 生成失败率 | <= 5% |
| 生成后进入编辑器比例 | >= 40% |

## 4. 目标用户

### 4.1 核心用户

| 用户类型 | 需求 | 痛点 |
| --- | --- | --- |
| 普通 Minecraft 玩家 | 想要独特皮肤 | 不会画像素画，不懂皮肤模板 |
| 服务器玩家/公会成员 | 需要统一阵营或职业皮肤 | 手工批量制作成本高 |
| 内容创作者/主播 | 需要符合人设的专属形象 | 找人定制慢，修改沟通成本高 |
| 家长/低龄玩家 | 想快速做一个安全、可爱的皮肤 | 工具太复杂，素材质量不稳定 |
| 皮肤创作者 | 需要快速出草图和变体 | 从零绘制耗时，灵感不足 |

### 4.2 典型场景

- 用户输入“蓝色卫衣、白发、赛博风、背后有小披风的男生”，生成皮肤并导出。
- 服务器管理员输入“中世纪骑士团，红黑配色，胸前有盾牌标志”，批量生成多套成员皮肤。
- 主播上传头像或角色设定图，生成 Minecraft 风格皮肤。
- 玩家生成后在编辑器中把鞋子改成黑色，把眼睛改成绿色。

## 5. 产品范围

### 5.1 MVP 范围

MVP 聚焦“从文字生成可用皮肤”，包含：

- 文本提示词生成 Minecraft 皮肤。
- 皮肤 2D 展开图展示。
- 3D 角色预览。
- 基础参数控制：性别/体型、风格、主色、服装类型、复杂度。
- 生成结果多方案展示。
- 基础像素编辑：画笔、橡皮、吸色、填充、撤销/重做。
- 导出 64x64 PNG。
- 历史记录和再次编辑。

### 5.2 非 MVP 范围

以下能力不进入首版，可作为后续版本：

- 多人协作编辑。
- 完整皮肤素材市场。
- Minecraft 账号直连上传。
- 服务器统一制服批量管理。
- 手机端深度编辑器。
- AI 视频或 3D 模型生成。

## 6. 核心用户流程

### 6.1 文生皮肤流程

1. 用户进入首页或生成页。
2. 输入角色描述。
3. 选择基础参数：模型、风格、主色、服装、配饰。
4. 点击生成。
5. 系统生成 2-4 个皮肤候选结果。
6. 用户查看 3D 预览并选择一个结果。
7. 用户进入编辑器做微调。
8. 用户导出 PNG 或保存到作品库。

### 6.2 图生皮肤流程

1. 用户上传参考图，如头像、角色设定、服装图片。
2. 系统识别主要元素：发色、服装、配色、配饰。
3. 用户补充文字要求。
4. 系统生成 Minecraft 皮肤候选。
5. 用户预览、编辑并导出。

### 6.3 局部修改流程

1. 用户在 3D 预览中点击头部、身体、手臂或腿部。
2. 用户输入局部修改要求，如“把帽子改成猫耳帽”。
3. AI 仅重绘对应区域，保留其他区域。
4. 用户确认或回退。

## 8. 页面需求

### 8.1 生成页

生成页是产品主入口，不做复杂营销页。首屏直接提供提示词输入和生成参数。

核心组件：

- 提示词输入框。
- 示例提示词按钮。
- 参数面板。
- 生成按钮。
- 生成进度状态。
- 候选结果网格。

关键状态：

- 空状态：展示 3-5 个可点击示例。
- 生成中：显示进度、预计时间和可取消按钮。
- 生成失败：说明失败原因并允许重试。
- 点数不足：提示登录或购买点数。

### 8.2 预览与编辑页

核心组件：

- 左侧工具栏：画笔、橡皮、吸色、填充、撤销、重做。
- 中间区域：3D 预览和 2D 展开图切换。
- 右侧面板：颜色、图层、AI 局部修改、导出。
- 底部版本条：显示历史版本。

### 8.3 作品库

核心组件：

- 作品列表。
- 搜索与筛选。
- 最近编辑时间。
- 复制、下载、删除。
- 从历史版本继续编辑。

## 9. AI 能力设计

### 9.1 输入理解

系统需要从用户输入中抽取：

- 角色类型：人类、骑士、法师、机器人、动物拟人等。
- 外观特征：发色、眼睛、肤色、表情。
- 服装：上衣、裤子、鞋子、盔甲、裙装等。
- 风格：可爱、暗黑、赛博、幻想、校园、复古等。
- 配色：主色、辅色、强调色。
- 禁止元素：用户不想出现的内容。

### 9.2 生成约束

AI 输出必须满足：

- 文件格式为 PNG。
- 皮肤尺寸为 64x64。
- 贴图布局符合 Minecraft 皮肤规范。
- 头、身体、手臂、腿部区域不能错位。
- 像素风格清晰，避免过度平滑和照片化。
- 透明区域只允许出现在合理的外层区域，基础身体区域必须完整。

### 9.3 质量检查

生成后应自动检查：

- PNG 尺寸是否正确。
- 关键身体区域是否为空。
- 左右肢体是否明显错位。
- 是否存在大面积透明异常。
- 是否符合安全内容策略。
- 是否与用户提示词明显不匹配。

### 9.4 推荐模型方案

MVP 可采用两段式架构：

1. 大语言模型解析用户需求，生成结构化皮肤描述。
2. 图像生成/像素生成模型根据结构化描述生成 64x64 皮肤贴图。

为保证 Minecraft 贴图结构稳定，建议使用以下策略：

- 使用标准皮肤 UV 模板作为生成约束。
- 对身体区域分块生成或分块校正。
- 对生成结果做规则校验和自动修复。
- 使用已有高质量皮肤数据进行风格微调或 LoRA 训练。

## 11. 非功能需求

### 11.1 性能

- 普通生成任务应在 60 秒内完成。
- 预览器首屏加载时间应小于 3 秒。
- 编辑器操作延迟应小于 100ms。
- 导出 PNG 应在 2 秒内完成。

### 11.2 兼容性

- 支持 Chrome、Edge、Firefox、Safari 最新两个主版本。
- 桌面端为主要体验。
- 移动端支持生成、预览和下载，复杂像素编辑可降级。

### 11.3 稳定性

- 生成任务需要队列和超时机制。
- 失败任务不扣除用户点数，或自动退回点数。
- 所有编辑操作需要本地临时保存，避免刷新丢失。

### 11.4 安全与合规

- 禁止生成仇恨、色情、暴力极端主义等违规内容。
- 对用户上传参考图进行安全检测。
- 防止生成明显侵犯第三方角色版权的内容，尤其是直接复刻知名 IP。
- 页面需要注明产品非 Mojang 或 Microsoft 官方产品。

## 12. 权限与风控

| 风险 | 处理方式 |
| --- | --- |
| 用户上传不当图片 | 上传前/后内容审核 |
| 用户要求生成知名 IP 角色 | 提醒改为原创相似风格，不直接复刻 |
| 批量滥用生成 | 登录、频控、点数限制 |
| 生成结果结构错误 | 自动校验，不合格结果重新生成或修复 |
| 用户作品丢失 | 自动保存、版本管理、本地缓存 |

## 14. 验收标准

### 14.1 生成能力验收

- 输入中文提示词后，系统能生成至少 2 个候选皮肤。
- 每个候选皮肤都是 64x64 PNG。
- 皮肤在 3D 模型中显示时，头、身体、手臂、腿部位置正确。
- 生成结果与提示词中的主要元素有明显对应关系。
- 生成失败时，用户能看到明确提示并重新生成。

### 14.2 编辑器验收

- 用户可以使用画笔修改任意像素。
- 用户可以撤销和重做编辑操作。
- 用户可以在 2D 和 3D 预览间切换。
- 用户保存后再次打开，编辑结果不丢失。

### 14.3 导出验收

- 用户点击导出后能下载 PNG 文件。
- 导出文件尺寸为 64x64。
- 导出的皮肤可以被 Minecraft 皮肤导入流程识别。
